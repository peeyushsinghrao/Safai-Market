# SAFAI MARKET — PHASE F ADDENDUM
# Missing Requirements & Future-Ready Architecture

---

# PURPOSE

This document contains all important requirements that are currently missing or only partially defined in the existing PRD, Audit Report, and Implementation Plan.

These requirements must be incorporated into the final architecture before production deployment.

---

# F1 — MULTI-BRANCH ARCHITECTURE

Current architecture supports:

- Multi Shop

Missing:

- Multi Branch

Example:

Shop:
Anupurna Traders

Branches:

- Main Market Branch
- Railway Road Branch
- Bus Stand Branch

---

## Database

Create:

```text
branches
```

Fields:

```text
id
shop_id
name
address
phone
manager_name
status
created_at
updated_at
```

---

## Product Support

Products may be:

- Shared across branches
OR
- Managed separately per branch

Future-ready architecture required.

---

## Stock Management

Stock must be tracked per branch.

Example:

Product A

Main Branch:
50 pcs

Railway Branch:
20 pcs

Bus Stand:
5 pcs

---

## Billing

Every bill must contain:

```text
shop_id
branch_id
```

---

## Reports

Reports must support:

- Shop Level
- Branch Level


within seconds.

---

```text

Accounting Software
Complex Business Suite
```

- WhatsApp
- PhonePe
- Blinkit

Simple UI

Powerful Backend

Fast Workflow
